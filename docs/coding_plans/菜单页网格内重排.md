# 菜单页 `in-grid reflow`（网格内重排）详细实施计划（含子代理审查结论吸收版）

## 摘要
本计划把菜单页聚焦效果收敛到**单一 `LazyVerticalGrid` 运行时架构**，不再依赖任何 `scene switch`（场景切换）。  
目标效果是：聚焦卡放大、周围卡整体缩放+位移+重排感、聚焦详情展开；当分类菜品超过 9 个时，使用**按行滚动**（`row-by-row scroll`），且重排只作用于当前可见 `3x3 viewport`（3x3 可视窗）。  
本版本已吸收子代理审查提出的 P0/P1 风险（架构残留、滚动语义、测试缺口）。

## 关键实现变更（决策完整）
1. 运行时架构收敛为单网格
- 删除右侧内容区按 `MenuScene` 分支渲染的路径，`MenuRoute` 右侧主体永远渲染 `MenuItemGrid`。
- 清理 `FocusStageLayout` 在菜单运行时路径的入口；该组件保留为历史参考，不参与当前交互链路。
- `Back` 行为保持为“Browse 根层 -> 上层回调”，不引入“先退 FocusScene”中间态。

2. 状态模型改为单事实源（`single source of truth`）
- `MenuUiState` 收敛为 Browse 语义：`selectedCategoryId + categories + browseSceneState`。
- 交互状态只保留：分类选择、网格焦点 `itemId`、行锚点（`rowIndex`）与恢复请求。
- 滚动恢复从 `item-level anchor`（`firstVisibleItemIndex+offset`）切换为 `row-level anchor`（`rowIndex`，恢复时 `index=rowIndex*3, offset=0`），统一读写口径。
- 与旧 `FocusSceneState` 相关的 reducer 路径在运行时移除，避免实现者二次判断“何时切场景”。

3. 网格内重排算法（`local reflow`）
- 新增 `pose resolver`（姿态解析器），输入为：`itemIndex`、`focusedIndex`、`visibleRowStart`、`columns=3`、`visibleRows=3`。
- 先做行对齐：`visibleRowStart = (firstVisibleItemIndex / 3) * 3`，再计算 `rowSlot`，避免非行对齐时 pivot/位移错误。
- 仅对可见 `3x3` 产出强动画：`scale/alpha/offset/zIndex/showExpandedDetails`。
- 可见窗外卡片不参与重排，仅保持轻量衰减，避免大列表 `jank`（卡顿）。
- 边界裁剪保留并强化：左/右/上/下边界禁止向外推挤，避免侵占分类栏与顶部文案区。

4. 动效令牌化（`motion tokens`）与默认参数
- 引入集中动效令牌，替代分散硬编码，参数可配置且有边界约束。
- 默认参数（首版）：
`focusedScale=1.24`
`near/mid/far scale=0.82/0.88/0.94`
`pushXRatio=0.24`
`pushYRatio=0.18`
`alpha near/mid/far=0.72/0.80/0.88`
- 约束边界：
`focusedScale` 限定在 `1.12~1.28`
`pushX<=0.26`
`pushY<=0.20`
`alpha>=0.70`。
- 主卡详情区只在焦点卡展开，失焦收拢，且不触发模式迁移。

5. 按行滚动控制（非按页）
- 明确实现 `row-by-row`：焦点位于可视窗底行且继续 `DirectionDown` 时，仅滚动 `+1 row`（索引 `+3`）；顶部同理 `-1 row`。
- 滚动后焦点恢复到同列目标卡，保证列对齐连续性。
- 不允许整页跳跃滚动（`page scroll`）作为默认行为。
- 继续保持 `BROWSE_GRID_USER_SCROLL_ENABLED=false`（仅遥控器 D-pad 驱动）。

6. 已渲染/未渲染混合进入可视窗的连续性
- `LazyGrid` 使用稳定 `key=itemId`，避免 item 复用导致动效串位。
- 动效状态外提到父层 `pose cache`（按 `itemId` 存储最近姿态），新进入可视窗卡片使用 `enter pose` 过渡到目标姿态，避免“突跳”。
- 分类切换时清理该分类的 `pose cache`，防止跨分类残影。

## 接口与类型变更（实现者必须按此落地）
1. 状态接口
- `MenuUiState` 删除 `scene` 与 `focusSceneState` 字段。
- 新增/替换 Browse 恢复请求类型为行语义请求（`rowIndex` 驱动），废弃 item+offset 恢复语义。

2. 纯函数接口
- `resolveBrowseCardFocusMotion(...)` 参数改为显式接收 `visibleRowStart`（已对齐）而不是直接依赖 `firstVisibleItemIndex`。
- 新增 `resolveBrowseRowScrollIntent(...)` 纯函数，返回 `NoScroll / ScrollUpOneRow / ScrollDownOneRow`，供按键处理与测试复用。
- `resolveBrowseFocusedCardScale(...)` 不再依赖 `FocusScene` 尺寸令牌，改为基于网格动效令牌计算。

## 测试计划与验收标准
1. 单元测试（`unit test`）
- 动效契约矩阵：首屏/中段/末段 + 左中右列焦点，断言 `scale/alpha/offset/pivot/zIndex`。
- 边界安全：左列不左推、顶行不上推、右列不右推、底行不下推。
- 行滚动意图：连续 `DirectionDown` 每次仅 `+3`，连续 `DirectionUp` 每次仅 `-3`，无整页跳跃。
- `visibleRowStart` 行对齐正确性：非对齐索引输入不导致 pivot 错位。

2. UI 测试（`androidTest`）
- “永不切场景”回归：交互过程中不存在 `focus-scene` 节点。
- >9 菜品真实 D-pad 路径：连续方向键下移触发按行滚动，第 10 张卡可达且滚动步长为 1 行。
- 聚焦详情展开可见性：仅焦点卡展示详情区；焦点切换后旧卡详情收拢。
- 顶部文案与左侧分类不被重排侵占（可见断言保持稳定）。

3. 验收标准（`acceptance criteria`）
- 全程单网格，无场景切换。
- 聚焦视觉明显接近设计稿静态截图风格（主卡突出、周卡收敛、重排感成立）。
- 超过 9 个菜品时严格按行滚动。
- 无越界侵占、无明显跳变、无焦点丢失。

## 假设与默认决策
- 本轮不删除 `FocusStageLayout` 文件本体，仅移出运行时链路，避免无关回归。
- 分类切换默认重置到 `rowIndex=0` 与该分类首焦点。
- 若数据刷新导致当前焦点菜品消失，回退到当前行首可用项；若当前行无可用项，回退到分类首项。
- 动效调参使用令牌集中管理；首版按上文默认值落地，后续仅改令牌不改算法。
