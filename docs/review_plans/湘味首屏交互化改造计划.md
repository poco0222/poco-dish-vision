# 湘味首屏交互化改造计划（含 Pencil MCP 设计对齐）

## Summary

- 将 `Home / Attract Screen`（首页吸引态）改为 `interactive hero carousel`（可交互主视觉轮播）：`Left/Right`（左右键）在首屏内循环切换 5 张推荐卡，右侧 `hero image`（主图）和左侧文案同步切换。
- 保留 `auto-play`（自动轮播）：默认开启；用户手动切卡后暂停；`10s idle timeout`（10 秒空闲超时）后从当前卡继续轮播；轮播间隔固定为 `5s`。
- `Up/Down/Center`（上下/确认键）都直接进入 `Browse`（菜单页）；`Menu key`（菜单键）进入 `Settings`（设置页）；`Browse -> Attract` 的现有 `15s idle timeout` 保持不变。
- 主题令牌 `theme tokens`（主题令牌）和首页 UI 结构落盘前，必须以 `Pencil MCP` 读取 [pencil-new.pen](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/design/pencil-new.pen) 为准，不凭截图肉眼硬编码；代码完成后再用同一设计稿做回看校验。

## Key Changes

- `Pencil MCP` 设计对齐流程：
  - 在实现前读取 [pencil-new.pen](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/design/pencil-new.pen) 的 `R3XHK`（湘味首屏）节点，使用 `batch_get / get_variables / get_screenshot` 提取首页所需的 `color / spacing / corner radius / typography / chip / card hierarchy`（颜色、间距、圆角、字阶、标签、卡片层级）。
  - 将 `Pencil` 中可复用的色板、圆角、间距映射到 [ColorTokens.kt](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/core/ui/src/main/java/com/poco/dishvision/core/ui/theme/ColorTokens.kt) 和 [Dimens.kt](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/core/ui/src/main/java/com/poco/dishvision/core/ui/theme/Dimens.kt)；无法一一映射的值，先以“最小必要 token”落盘，并在注释里标明来源是 `pencil-new.pen`。
  - 完成实现后再次对照 `Pencil MCP screenshot`（设计稿截图）校验 Home 首屏结构、视觉层级和关键间距，不要求像素级完全一致，但要保证品牌区、左文案区、右主图区、底部 5 卡带的关系一致。

- `feature/home`：
  - 扩展 [HomeUiState.kt](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/feature/home/src/main/java/com/poco/dishvision/feature/home/HomeUiState.kt)，从“标题+副标题+推荐列表”升级为 Home 专用 `presentation model`（展示模型），包含品牌头部、标签组、5 张首屏卡、轮播配置。
  - 新增 `HomeShowcaseItem` 与 `HomeCarouselController`：前者承载 `hero` 与卡片所需的纯展示字段；后者作为纯 Kotlin 状态机，负责当前索引、左右循环、自动轮播、手动暂停与 10 秒恢复。
  - 重写 [HomeRoute.kt](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/feature/home/src/main/java/com/poco/dishvision/feature/home/HomeRoute.kt) 与首页 UI 组合，版式以 `Pencil MCP` 读取结果为准对齐 `湘味首屏`。
  - 首页卡片使用 `visual focus`（视觉聚焦）而不是系统真实焦点；用高亮边框、底色和文案切换表达“当前聚焦卡”，避免自动轮播与真实 TV `focus`（焦点）冲突。
  - 首屏图片使用本地资源；资源命名和顺序与 `Pencil` 首屏 5 张展示卡保持一一对应。

- `app/navigation`：
  - 调整 [AppNavHost.kt](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/app/src/main/java/com/poco/dishvision/navigation/AppNavHost.kt) 的按键分工：Home 不再把任意方向键都当作进入 `Browse`；`Left/Right` 交给 Home 自己消费，`Up/Down/Center` 进入 `Browse`，`Menu key` 进入 `Settings`。
  - 移除首页右上角可见 `设置` 按钮；设置入口统一改为遥控器 `Menu key`，避免破坏设计稿右上角品牌标签。
  - 保留现有 `Back`（返回键）与 `BrowseModeController` 的返回链路：`Settings -> Browse -> Attract/Home`。

- `core/ui`：
  - 更新共享主题入口 [PocoTheme.kt](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/core/ui/src/main/java/com/poco/dishvision/core/ui/theme/PocoTheme.kt) 及相关通用容器，使首屏从当前冷蓝灰 `glass UI`（玻璃化界面）转向“暖暗底 + 辣椒红 + 铜金强调”的 Xiangwei 风格。
  - 共享 token（设计令牌）命名保持可复用，不把值写死成 `home-only`；后续 `Browse / Detail` 可直接沿用。

## Test Plan

- 单元测试：
  - 验证 `Left/Right` 循环切换。
  - 验证手动切卡后自动轮播暂停。
  - 验证 10 秒无操作后从当前卡恢复自动轮播。
  - 验证自动轮播每 5 秒推进一张。
- Home UI 测试：
  - 首屏默认渲染 `新华饭店`、3 个标签、5 张卡和第一张主图。
  - `Right` 后主图、标题、卡片高亮切到下一张；`Left` 在第一张时回到最后一张。
  - `Up`、`Down`、`Center` 任一按键都能触发进入 `Browse` 的回调。
- Navigation smoke test`（导航冒烟测试）`：
  - 应用启动进入 `home-screen`。
  - `Menu key` 从 Home 进入 `settings-screen`。
  - 从 `Settings` 返回进入 `browse-screen`。
  - 从 `Browse` 返回回到 `home-screen`。
- 设计对齐验证：
  - 用 `Pencil MCP get_screenshot` 再看一次 [pencil-new.pen](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/design/pencil-new.pen) 的 `R3XHK`。
  - 对照实现后的 Home：确认品牌位置、主图比例、左侧两行主标题、标签组、底部 5 卡数量与层级关系一致。
  - 若 token（设计令牌）和设计稿有偏差，以 `Pencil` 读出来的值回修代码，不靠主观目测。

## Assumptions

- 本轮不修改 `catalog.json` 与 `Browse` 菜单数据语义；首页使用独立的湘味展示数据。
- `Browse` 进入后仍定位到默认分类，不根据首屏当前卡跳分类或跳菜品。
- 首页离开后再次回到 Home，默认重置到第 1 张卡并重新进入自动轮播；“从当前卡恢复轮播”只适用于用户仍停留在同一轮 Home 会话内的 10 秒空闲恢复。
- `Center` 与 `Up/Down` 行为保持一致，均进入 `Browse`；首页不新增“直接开菜品详情”的独立导航分支。
- 所有主题令牌和首页关键 UI 比例，以 [pencil-new.pen](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/design/pencil-new.pen) 的 `Pencil MCP` 读取结果为实现依据。
