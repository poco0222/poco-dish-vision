## Key Changes
- 以 [pencil-new.pen](/Users/PopoY/workingFiles/Projects/POCO/POCO-DISH-VISION/design/pencil-new.pen) 的 `SVMLB` 和 `npcH9` 为唯一 `source of truth`（事实来源）；旧文档里的 `detail dock`（详情停靠条）不再反向约束这轮菜单布局。
- 将菜单拆成两个明确 `scene`（场景）：
  - `BrowseScene` 对齐 `SVMLB`：左分类导轨 + 右标题区 + 固定三列、可纵向滚动的菜品网格。
  - `FocusScene` 对齐 `npcH9`：保留现有 `FocusStageLayout` 思路，但作为独立状态切换，不再嵌在浏览态右栏内部做混合过渡。
- 菜单分类复用首屏卡带的 `card motion`（卡片动效）和 `visual focus`，但只复用动画机制，不复用具体卡体尺寸：
  - 分类项增加 `spring scale`（弹簧缩放）、弱光扩散 `glow`（辉光）、`zIndex` 抬升、边框/底色联动。
  - 分类项仍保持设计稿的 20px 圆角、14/18 内边距、红底/金边/米白字；这套效果只用于分类，不扩散到菜品卡。
- 分类切换动画以“分类卡切换”本身为主，不复用首屏 `hero wipe`（主图区擦除）：
  - 左侧分类项使用首屏卡带同款动效。
  - 右侧标题和菜品区只保留轻量 `Crossfade/AnimatedContent`（淡入淡出/内容切换），不做 `hero/copy` 区域那套 wipe。
- 比例布局统一收口到菜单页全链路：
  - 继续使用 `ScreenProportions` 承接外层宽高、起点坐标、主列宽度。
  - 把所有会影响“挤压感”的浏览态/聚焦态大中型尺寸迁入 `ScreenProportions`：分类项高度感、菜品卡图高、卡体内边距、网格视口高度、聚焦态内部留白。
  - `corner radius`（圆角）、`border width`（边框宽度）、glow blur 等 `micro token`（微型令牌）继续留在 `Dimens`。
- 浏览态按“纵向滚动网格”落地：
  - 保持 3 列、顶部对齐、首屏观感尽量接近 `SVMLB` 的 9 卡布局。
  - 分类切换时网格回到该分类顶部。
  - 从 `FocusScene` 返回时恢复该分类上次滚动位置和上次聚焦菜品，而不是永远跳回第一张。
- 状态层只做 UI 级整理，不改 `Repository` / `catalog schema`（目录数据结构）：
  - 增加分类切换动画所需的方向/索引 `bookkeeping`（状态记录）。
  - 为每个分类维护独立 grid scroll/focus restore 状态。
  - 恢复 `BrowseModeController` 的 `15s idle timeout` 语义，让实现和测试重新一致。
