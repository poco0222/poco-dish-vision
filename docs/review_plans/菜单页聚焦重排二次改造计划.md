# 菜单页聚焦重排二次改造计划

## Summary
- 保持 Browse 页 `3x3` 基线、单场景交互和现有 `row-by-row scroll`（逐行滚动）语义不变。
- 视觉方向采用 `virtual reflow`（虚拟重排），强度接近你认可的 `B3`，但不进入 `mini stage`（迷你舞台）那种独立舞台体量。
- 聚焦后的稳态必须一眼看出“主卡已成为主视觉，其他卡已缩小让位”；`stacking/push`（堆挤/推挤）只保留为过渡动画，不再是最终状态。
- 稳态规则固定为：底部辅助卡必须同一水平线；顶部辅助卡允许轻微错层；中间卡聚焦时可四向扩张，但不能侵占左侧导轨和顶部文案安全区。

## Key Changes
- 主要改造集中在 `feature/menu/src/main/java/com/poco/dishvision/feature/menu/MenuItemGrid.kt`。
- 把当前基于 `scale + offset ratio`（缩放 + 位移比例）的姿态模型，升级为基于 `target frame`（目标框）的布局模型。
- 新增内部布局目标类型，例如 `BrowseCardLayoutTarget`，至少包含：`x/y/width/height/alpha/zIndex/pivot/showExpandedDetails`。
- 将当前“固定 `LazyVerticalGrid` 槽位，再叠加平移”的稳态方案，替换为“仅对当前可见 `3x3 viewport`（3x3 可视窗）计算真实目标框”的方案。
- 保留现有 Browse 状态事实源和焦点恢复语义，不改 `MenuRoute` / ViewModel 的外部行为；仍然以 `focusedItemId + rowIndex` 驱动恢复和逐行滚动。
- `center focus`（中心卡聚焦）规则明确为：
  - 主卡四向扩张，但体量介于当前浏览器稿 `B1` 与 `B3` 之间，最终更靠近 `B3`。
  - 中间行左右两张邻卡是第一层让位对象，缩小和横向退让最明显。
  - 底部三张卡只允许缩小和横向重排，不允许破坏统一基线。
  - 顶部三张卡允许轻微错层，但错层只作为“呼吸感”补偿，不能做成凌乱漂浮。
- `edge focus`（边界卡聚焦）规则沿用并强化当前方向约束：
  - 左边界聚焦不得侵占分类导轨。
  - 顶边界聚焦不得侵占标题/描述区。
  - 右边界和底边界保持同样的单向扩张安全规则。
- 焦点详情区继续只挂在聚焦卡上，但尺寸和展开动画要跟随新的 `target frame`，不再依赖旧的纯 `graphicsLayer scale` 放大逻辑。
- 当前 `resolveBrowseFocusPushOffsets(...)` 这类“比例位移”辅助函数应降级为过渡动画辅助，或直接被新的 `target frame transition`（目标框过渡）替代。

## Interfaces And Behavior
- 重要内部接口变更：
  - `resolveBrowseCardFocusMotion(...)` 需要升级为“解析目标框 + 过渡参数”的接口，而不只是返回 `scale/alpha/offset`。
  - `BrowseCardFocusMotion` 如果保留，应扩展到能表达真实布局目标；如果表达力不够，直接拆成“稳态布局目标 + 过渡动画参数”两层。
- 不做的事：
  - 不回到 `FocusScene`（独立焦点场景）。
  - 不把 Browse 稳态改成固定舞台布局。
  - 不改变 `GRID_COLUMNS = 3`、`GRID_VISIBLE_ROWS = 3`、描述一行裁切等已确认的 Browse 基线。

## Test Plan
- 更新 `feature/menu/src/test/java/com/poco/dishvision/feature/menu/MenuItemGridFocusMotionTest.kt`：
  - 覆盖中心卡聚焦时的目标框几何。
  - 断言左右中邻是第一层让位对象。
  - 断言底部三张卡在中心聚焦时共享同一 `Y` 基线。
  - 断言顶部错层存在上限，不会无限漂移。
  - 保留并继续验证边界聚焦不得越界的现有契约。
- 更新尺寸/布局类单测：
  - 未聚焦态仍满足 `3x3` Browse 基线。
  - 聚焦态目标框必须落在可视窗内，并避开左导轨和顶部文案安全区。
- 更新 `feature/menu/src/androidTest/java/com/poco/dishvision/feature/menu/BrowseLayoutContractTest.kt` 或相邻 UI 测试：
  - 验证全程不出现 `FocusScene` 节点。
  - 验证中心卡聚焦时底部辅助卡的视觉基线一致。
  - 验证标题区、辅助文案区、分类导轨区不被新重排侵占。
  - 验证超过 9 个菜品时仍然是逐行滚动，而不是整页跳动。
- 验收前必须做一次设备或 `emulator`（模拟器）截图对照，重点核对“中心卡聚焦”和“边界卡聚焦”两种稳态，而不是只看单元测试通过。

## Assumptions
- 已确认的视觉方向是 `virtual reflow`，不是 `token-only boost`，也不是完整 `mini stage`。
- 中心卡聚焦示意页中的整体关系已被接受，可作为实现参照。
- 底部齐平是硬规则；顶部轻微错层是软规则，只能小幅使用。
- 本轮目标是把“重排是稳态、堆挤只是过程”真正落实到布局模型，而不是继续只靠把现有参数调大。
