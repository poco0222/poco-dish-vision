## ⚠️ 强制规则（最高优先级）

> **以下规则具有最高优先级，必须严格遵守，违反任何一条都是不可接受的。**

### 规则 1: 回复要求

你的回复**必须**用中文, 专业术语**必须**用英文(最好能带上翻译)

### 规则 2：代码注释要求

编写的所有代码**必须**包含适当的注释(author等编写者必须是PopoY)，包括但不限于：

| 注释级别   | 要求                                 | 示例                                            |
| ---------- | ------------------------------------ | ----------------------------------------------- |
| **文档级** | 文件顶部说明文件用途、作者、创建日期 | `/** @file 用户认证模块 */`                     |
| **方法级** | 函数/方法上方说明功能、参数、返回值  | `/** @param userId 用户ID @returns 用户信息 */` |
| **变量级** | 复杂变量或常量旁说明用途             | `// 最大重试次数`                               |
| **逻辑级** | 复杂业务逻辑处说明意图               | `// 先验证权限，再执行操作`                     |

### 规则 3：遵循官方最佳实践

你输出的所有计划以及改动或新增的所有代码**必须**符合相应技术栈的官方最佳实践

### 规则 4：git提交信息

生成的提交信息除了专业术语外, 都尽量使用中文
